# Telegram-бот для изучения УК РФ

Готовый бот на `python-telegram-bot` с SQLite, карточками, ревизией, статистикой, настройками и ежедневными напоминаниями.

## Что умеет

- Главное меню с инлайн-кнопками
- Выбор одной или нескольких глав УК РФ
- Обучение и ревизия с отдельным подбором карточек
- Несколько форматов ответа:
  - обычные карточки
  - тест с вариантами ответа
  - верно / неверно
  - смешанный режим
- Флеш-карточки с подсказками, показом ответа, кнопками `Знаю` / `Не знаю` / `Пропустить`
- Ситуационные задачи как в формате обычной карточки, так и в формате теста
- Повторение карточек до статуса `выучена`
- Прогресс, баллы, серии правильных ответов и статистика по главам
- Настройки: режим по умолчанию, смешанный режим, варианты ответа, размер сессии, подсказки, напоминания, сброс прогресса
- SQLite-хранилище пользователей, карточек, прогресса, истории ответов и текущей сессии

## Структура проекта

- `app.py` — точка входа
- `bot.py` — Telegram-логика и inline UI
- `database.py` — SQLite-схема, миграции и бизнес-логика прогресса
- `config.py` — загрузка настроек из `.env`
- `data/cards.json` — стартовый набор карточек

## Быстрый запуск

1. Создайте и активируйте виртуальное окружение.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте `.env` на основе `.env.example` и укажите токен Telegram-бота.
4. Запустите:

```bash
python app.py
```

Для Windows добавлен быстрый запуск через `run_bot.bat`.

## Переменные окружения

- `TELEGRAM_BOT_TOKEN` — токен бота от BotFather
- `BOT_TIMEZONE` — таймзона для ежедневных напоминаний
- `DB_PATH` — путь до SQLite-файла
- `MASTERED_STREAK` — сколько правильных ответов подряд нужно для статуса `выучена`

## Формат карточек

Поддерживаются типы:

- `flashcard`
- `multiple_choice`
- `true_false`
- `scenario`

Если у карточки нет поля `type`, она автоматически считается `flashcard`.

Пример:

```json
{
  "chapter_code": "21",
  "chapter_title": "Преступления против собственности",
  "article": "Ст. 158 УК РФ",
  "type": "multiple_choice",
  "question": "Какая статья УК РФ относится к краже?",
  "answer": "Ст. 158 УК РФ — кража",
  "hint": "Это преступление против собственности",
  "explanation": "Кража регулируется статьей 158 УК РФ",
  "difficulty": "easy",
  "options": [
    "Ст. 105 УК РФ",
    "Ст. 158 УК РФ",
    "Ст. 111 УК РФ",
    "Ст. 163 УК РФ"
  ],
  "correct_option": 1
}
```

Для `true_false` можно использовать `correct_bool`, а для старых карточек достаточно `question`, `answer`, `chapter_code`, `chapter_title`, `article`.

## Логика ревизии

Ревизия подбирает карточки по пользователю и выбранным главам, если карточка уже встречалась в обучении:

- была отвечена хотя бы один раз
- была с ошибками
- находится в статусе `learning`
- уже была `mastered`, но пользователь хочет повторение

Если подходящих карточек нет, бот показывает понятное сообщение:

- `Для ревизии пока нет карточек`
- `Сначала пройдите обучение, чтобы появились карточки для повторения`

## Модель данных

### `users`

- Telegram user ID
- выбранные главы
- настройки сессии и подсказок
- настройки режима обучения
- напоминания
- баллы и серии

### `cards`

- глава
- статья
- тип карточки
- вопрос
- ответ
- подсказка
- пояснение
- сложность
- варианты ответа

### `user_card_progress`

- статус карточки (`new`, `learning`, `mastered`)
- количество правильных подряд
- число правильных и неправильных ответов

### `answer_history`

- пользователь
- карточка
- дата и время
- результат ответа
- начисленные очки

### `session_state`

- активный режим (`study` / `review`)
- формат обучения
- текущая карточка
- сериализованные варианты ответа после перемешивания
- очередь оставшихся карточек

## Важно

Стартовый набор карточек в `data/cards.json` демонстрационный. Перед продакшен-использованием его стоит расширить и проверить на юридическую точность.
