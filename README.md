# Telegram-бот для изучения УК РФ

Готовый бот на `python-telegram-bot` с SQLite, карточками, ревизией, статистикой, настройками и ежедневными напоминаниями.

## Что умеет

- Главное меню с современным inline UI и быстрым доступом к основным действиям
- Выбор одной или нескольких глав УК РФ
- Обучение и ревизия с отдельным подбором карточек
- Несколько форматов ответа:
  - обычные карточки
  - тест с вариантами ответа
  - верно / неверно
  - смешанный режим
- Флеш-карточки с подсказками, показом ответа, кнопками `Знаю` / `Не знаю` / `Пропустить`
- Ситуационные задачи как в формате обычной карточки, так и в формате теста
- Повторение карточек до статуса `выучена`
- Прогресс, баллы, серии правильных ответов и статистика по главам
- Настройки: режим по умолчанию, смешанный режим, варианты ответа, размер сессии, подсказки, напоминания, сброс прогресса
- Экран помощи с кратким объяснением режимов и навигации
- SQLite-хранилище пользователей, карточек, прогресса, истории ответов и текущей сессии

## Интерфейс и UX

Бот оформлен в едином стиле и рассчитан на короткие ежедневные сессии:

- Главное меню показывает краткий summary: прогресс за день, количество карточек для ревизии и число выбранных глав
- Выбор глав отображает текущий список фильтра и понятные маркеры `✅` / `◻️`
- Карточка показывает прогресс по сессии, главу, статью, тип, сложность и текущий статус изучения
- После ответа бот сразу показывает результат, правильный ответ, короткое пояснение и что произошло с карточкой
- Статистика разбита на блоки: общий прогресс, результаты, цель на сегодня и прогресс по главам
- Настройки собраны в компактный экран без перегрузки
- Экран помощи объясняет, как начать обучение, как работает ревизия и чем отличаются режимы

## Структура проекта

- `app.py` — точка входа
- `bot.py` — Telegram-логика и inline UI
- `database.py` — SQLite-схема, миграции и бизнес-логика прогресса
- `config.py` — загрузка настроек из `.env`
- `data/cards.json` — основной набор карточек, отсортированный по главам и статьям
- `data/cards_navigation.md` — навигационный индекс по главам и статьям для удобного редактирования

## Быстрый запуск

1. Создайте и активируйте виртуальное окружение.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте `.env` на основе `.env.example` и укажите токен Telegram-бота.
4. Запустите:

```bash
python app.py
```

Для Windows добавлен быстрый запуск через `run_bot.bat`.

## Переменные окружения

- `TELEGRAM_BOT_TOKEN` — токен бота от BotFather
- `BOT_TIMEZONE` — таймзона для ежедневных напоминаний
- `DB_PATH` — путь до SQLite-файла
- `MASTERED_STREAK` — сколько правильных ответов подряд нужно для статуса `выучена`

## Формат карточек

Поддерживаются типы:

- `flashcard`
- `multiple_choice`
- `true_false`
- `scenario`

Если у карточки нет поля `type`, она автоматически считается `flashcard`.

Пример:

```json
{
  "chapter_code": "21",
  "chapter_title": "Преступления против собственности",
  "article": "Ст. 158 УК РФ",
  "type": "multiple_choice",
  "question": "Какая статья УК РФ относится к краже?",
  "answer": "Ст. 158 УК РФ — кража",
  "hint": "Это преступление против собственности",
  "explanation": "Кража регулируется статьей 158 УК РФ",
  "difficulty": "easy",
  "options": [
    "Ст. 105 УК РФ",
    "Ст. 158 УК РФ",
    "Ст. 111 УК РФ",
    "Ст. 163 УК РФ"
  ],
  "correct_option": 1
}
```

Для `true_false` можно использовать `correct_bool`, а для старых карточек достаточно `question`, `answer`, `chapter_code`, `chapter_title`, `article`.

## Как добавить свои карточки

Бот загружает карточки из файла [data/cards.json](/C:/Users/acer/Desktop/zakonnik/data/cards.json) при запуске.

Для удобства редактирования:

- [data/cards.json](/C:/Users/acer/Desktop/zakonnik/data/cards.json) отсортирован по `chapter_code`, затем по `article`
- [data/cards_navigation.md](/C:/Users/acer/Desktop/zakonnik/data/cards_navigation.md) показывает быстрый индекс по главам и статьям
- комментарии внутрь JSON не добавляются, потому что обычный JSON их не поддерживает и бот не сможет прочитать файл

Если вы хотите добавить свой набор карточек:

1. Подготовьте JSON-массив карточек.
2. Добавьте его в [data/cards.json](/C:/Users/acer/Desktop/zakonnik/data/cards.json).
3. Либо положите отдельный файл с массивом карточек в папку `data/`, а затем объедините его с основным файлом.
4. Перезапустите бота командой `python app.py`.

Бот автоматически:

- добавляет новые карточки в SQLite;
- обновляет существующие карточки, если совпадают `chapter_code + article + question`;
- не требует ручной миграции базы для обычного добавления карточек.

### Обязательные поля

У каждой карточки должны быть:

- `chapter_code`
- `chapter_title`
- `article`
- `question`
- `answer`

### Рекомендуемые поля

- `type`
- `hint`
- `explanation`
- `difficulty`
- `example`
- `options`
- `correct_option`

### Поддерживаемые типы

- `flashcard`
- `multiple_choice`
- `true_false`
- `scenario`

Если поле `type` не указано, карточка считается `flashcard`.

### Формат по типам

`flashcard`:

```json
{
  "chapter_code": "21",
  "chapter_title": "Преступления против собственности",
  "article": "Ст. 158 УК РФ",
  "type": "flashcard",
  "question": "Какая статья УК РФ предусматривает ответственность за кражу?",
  "answer": "Ст. 158 УК РФ — кража",
  "hint": "Это тайное хищение чужого имущества",
  "explanation": "Кража предусмотрена статьёй 158 УК РФ.",
  "difficulty": "easy"
}
```

`multiple_choice`:

```json
{
  "chapter_code": "16",
  "chapter_title": "Преступления против жизни и здоровья",
  "article": "Ст. 111 УК РФ",
  "type": "multiple_choice",
  "question": "Какая статья УК РФ относится к умышленному причинению тяжкого вреда здоровью?",
  "answer": "Ст. 111 УК РФ — умышленное причинение тяжкого вреда здоровью",
  "hint": "Речь о тяжком вреде, причинённом умышленно",
  "explanation": "Ответственность за умышленное причинение тяжкого вреда здоровью установлена статьёй 111 УК РФ.",
  "difficulty": "easy",
  "options": [
    "Ст. 109 УК РФ",
    "Ст. 111 УК РФ",
    "Ст. 115 УК РФ",
    "Ст. 118 УК РФ"
  ],
  "correct_option": 1
}
```

`true_false`:

```json
{
  "chapter_code": "17",
  "chapter_title": "Преступления против свободы, чести и достоинства личности",
  "article": "Ст. 128.1 УК РФ",
  "type": "true_false",
  "question": "Статья 128.1 УК РФ посвящена клевете.",
  "answer": "Верно",
  "hint": "Это преступление против чести и достоинства личности",
  "explanation": "Клевета предусмотрена статьёй 128.1 УК РФ.",
  "difficulty": "easy",
  "correct_option": 0
}
```

`scenario`:

```json
{
  "chapter_code": "21",
  "chapter_title": "Преступления против собственности",
  "article": "Ст. 162 УК РФ",
  "type": "scenario",
  "question": "Нападавший, угрожая ножом, требует у прохожего телефон и деньги. Какая статья может применяться?",
  "answer": "Ст. 162 УК РФ — разбой",
  "hint": "Обрати внимание на нападение и опасное насилие либо угрозу его применения",
  "explanation": "Нападение в целях хищения с применением опасного насилия или угрозой такого насилия образует разбой по статье 162 УК РФ.",
  "difficulty": "medium"
}
```

### Важные правила

- В корне файла должен быть один JSON-массив: `[ { ... }, { ... } ]`
- Для `multiple_choice` должно быть ровно 4 варианта в `options`
- `correct_option` хранит индекс правильного варианта, начиная с `0`
- Для `true_false` обычно используется `correct_option: 0` для `Верно` и `correct_option: 1` для `Неверно`
- Если карточка уже существует с тем же `chapter_code`, `article` и `question`, она будет обновлена
- Если вопрос новый, карточка будет добавлена как новая

## Логика ревизии

Ревизия подбирает карточки по пользователю и выбранным главам, если карточка уже встречалась в обучении:

- была отвечена хотя бы один раз
- была с ошибками
- находится в статусе `learning`
- уже была `mastered`, но пользователь хочет повторение

Если подходящих карточек нет, бот показывает понятное сообщение:

- `Для ревизии пока нет карточек`
- `Сначала пройдите обучение, чтобы появились карточки для повторения`

## Модель данных

### `users`

- Telegram user ID
- выбранные главы
- настройки сессии и подсказок
- настройки режима обучения
- напоминания
- баллы и серии

### `cards`

- глава
- статья
- тип карточки
- вопрос
- ответ
- подсказка
- пояснение
- сложность
- варианты ответа

### `user_card_progress`

- статус карточки (`new`, `learning`, `mastered`)
- количество правильных подряд
- число правильных и неправильных ответов

### `answer_history`

- пользователь
- карточка
- дата и время
- результат ответа
- начисленные очки

### `session_state`

- активный режим (`study` / `review`)
- формат обучения
- текущая карточка
- сериализованные варианты ответа после перемешивания
- очередь оставшихся карточек

## Важно

Стартовый набор карточек в `data/cards.json` демонстрационный. Перед продакшен-использованием его стоит расширить и проверить на юридическую точность.
